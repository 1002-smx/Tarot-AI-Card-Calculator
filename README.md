# Tarot AI Card Calculator
基于 DeepSeek 大模型的本地 AI 塔罗解牌工具，自带 API Key 加密保护，跨平台、简洁易用、支持多轮追问。
制作这个项目是由于之前因为感情原因算牌，发现那个软件抽完牌算牌要89rmb，算牌的还是AI，于是我制作了此项目。

## 功能特性
- 完整 78 张塔罗牌库，自动随机抽牌并识别正逆位
- 情感抽 5 张，事业抽 6 张，今日运势抽 3 张，自定义问题抽 6 张
- AI 流式逐字输出，阅读体验更流畅
- API Key 本地安全存储，不暴露明文 （仅限加密版本）
- 临时文件存入系统隐藏目录，读完立即删除
- 支持多轮上下文追问，可补充线索或问题
- 同时提供加密版与非加密版
- 提供一键编译脚本（Windows），轻松打包 exe

## 使用建议
- 首次建议先运行非加密版本，生成并备份好你的密钥文件（.SAF）
- 确认保存无误后，再使用加密版本（.SEP）日常使用
- 加密文件如丢失或损坏，可重新打开非加密版生成的SAF文件找回密钥

## 关于“加密”的真实说明（重要）

本项目加密版（`main-encryption.py`）使用的加密方式为**字节取反（255 - byte）**，**不是**真正的密码学加密（如 AES）。

### 这个加密能做什么？

✅ **防止普通用户误读**：用记事本打开 `.SEP` 文件看到的是乱码，不会一眼看到 API Key 明文  
✅ **防止误删除**：文件名 `API.SEP` 不显眼，减少被当作垃圾文件清理的概率  
✅ **提供心理安全感**：让非技术用户不会因为看到明文而感到不安  
✅ **满足“本地不存明文”的基本要求**：持久化文件不是原始字符串

### 这个加密不能做什么？

❌ **防不了看过源码的人**：任何人都可以用 `255 - byte` 秒解密  
❌ **防不了恶意软件**：批量扫描时，取反是最容易识别的混淆之一  
❌ **防不了专业破解**：这不是密码学，只是字符变换

### - 不过真的会有人偷这个东西吗？

坦白说：**概率极低**。

- 这是你自己的 DeepSeek API Key，不是别人的
- DeepSeek 新用户有免费额度，即便被盗刷损失也很小（几块钱）
- 没有黑市专门收 DeepSeek Key，黑客懒得针对
- 最大的风险其实是**你自己不小心把明文 Key 截图发到网上**

所以这个“假加密”在实战中可能真的够用。

### 那你为什么还用？

因为**这个项目的威胁模型是**：

- 用户是普通人，不是黑客
- API Key 是用户自己的，价值很低
- 开源项目里，任何“真加密”都需要用户输入密码，会破坏体验
- 真正的风险不是“有人专门偷你的 Key”，而是“不小心把明文发给别人”

### 如果你真的担心安全

- **方案一**：使用本项目的**非加密版**（`main.py`），自己把 `API.SAF` 文件存放在**加密U盘**或**密码管理器**中
- **方案二**：在 [DeepSeek 控制台](https://platform.deepseek.com/) 设置 **API Key 每日限额**（最有效的防护）
- **方案三**：定期重置你的 API Key

### 结论

这个加密是**防君子不防小人**的“提示锁”，不是保险柜。  
它的目标是：**让普通用户不会无意中泄露 Key**，而不是对抗有意的攻击。

不过真的会有人偷这个东西吗？——大概率不会。所以放心用吧。行修改源码或使用非加密版 + 外部加密工具。

## 使用前
本项目依赖openai模块，请先安装openai模块
```bash
pip install openai
```

## 运行非加密版
```bash
python main.py
```

## 运行加密版
```bash
python main-encryption.py
```

## 编译非加密版 (Windows)：
```cmd
make_build_no_enc.bat
```

## 编译加密版 (Windows)：
```cmd
make_build_enc.bat
```

Windows 编译前请确保 pip 可用！编译后请在 dist 目录里找到 exe 文件！

## 安全说明
- 非加密版：API Key 明文存储在 API.SAF
- 加密版：API Key 字节加密存储在 API.SEP
- 明文仅临时存放在系统临时目录，读取完毕后立即删除
- 密钥仅存在本地，不会上传
